这份名为《A Scalable and Production Ready Sky and Atmosphere Rendering Technique》的论文（Epic Games提出，已应用于虚幻引擎）提供了一个非常适合落地的、高性能且支持动态变化的大气渲染方案。

根据您的要求，我将该文献提炼为一个**可落地的工程架构方案**。现阶段我们忽略繁琐的数学积分和微调参数，只关注**整体渲染管线（Pipeline）**和**各个阶段的核心工程任务**。

---

### 第一部分：整体渲染流程 (Rendering Pipeline)

整个大气渲染方案的核心思想是：**“将复杂的物理计算降维并缓存到多张分辨率较低的查找表（LUT）中，然后在实际渲染场景时对这些LUT进行廉价的采样。”**

由于该方案支持**全动态**（太阳移动、天气变化即时生效），以下流程需要在每一帧（或当天气/时间参数发生变化时）按顺序执行。

**管线执行顺序图：**
1. **全局参数准备** (CPU侧) -> 提交给GPU
2. **Pass 1: 生成/更新 透射率查找表 (Transmittance LUT)** -> 2D纹理
3. **Pass 2: 生成/更新 多重散射查找表 (Multi-scattering LUT)** -> 2D纹理 *(依赖Pass 1)*
4. **Pass 3: 生成/更新 天空视图查找表 (Sky-View LUT)** -> 2D纹理 *(依赖Pass 1, 2)*
5. **Pass 4: 生成/更新 空间透视体积图 (Aerial Perspective Volume LUT)** -> 3D纹理 *(依赖Pass 1, 2)*
6. **Pass 5: 最终场景合成 (Final Compositing)**
   - 渲染天空盒（使用 Pass 3 结果）
   - 渲染场景中物体的大气雾效（使用 Pass 4 结果）

### 第二部分：各流程步骤的具体工程任务 (Step-by-Step Implementation)

#### 1. 全局参数准备 (Data Setup)
*   **要做的事**：在CPU端组织物理大气所需的参数，并打包成一个常量缓冲区（Constant Buffer / UBO）发送给GPU。
*   **包含内容**：
    *   **几何数据**：行星半径（如地球）、大气层顶部半径。
    *   **介质属性**：瑞利散射（Rayleigh）、米氏散射（Mie）和臭氧（Ozone）的吸收系数与散射系数，以及它们随高度变化的密度分布参数。
    *   **光源数据**：主光源（太阳/月亮）的方向、强度、颜色。
    *   **视角数据**：当前相机的位置。

#### 2. Pass 1: 生成透射率查找表 (Transmittance LUT)
*   **要做的事**：计算光线从大气层内任意高度触发，穿过大气层到达太空时的光衰减程度。
*   **工程实现**：
    *   **目标**：一张低分辨率的 2D 纹理（如 `256 x 64`）。
    *   **计算方式**：启动一个 Compute Shader 或 Full-screen Pixel Shader。根据像素坐标映射出“当前高度”和“视线天顶角”。沿着这条射线步进（Ray-marching），累加沿途的大气吸收和散射造成的能量流失。

#### 3. Pass 2: 生成多重散射查找表 (Multi-scattering LUT)
*   **要做的事**：这是这篇论文的核心创新。用极为低廉的成本近似模拟光线在大气中无限次反弹的效果（取代了以前极其昂贵的多次迭代方法）。
*   **工程实现**：
    *   **目标**：一张极小分辨率的 2D 纹理（如 `32 x 32`）。
    *   **计算方式**：基于等比数列求和的近似法（O(1)复杂度）。在 Shader 中，对当前点周围的球面空间进行均匀采样（如64个方向），计算一次光线反弹的能量，然后套用论文中的几何级数公式直接得出无限次散射的近似值。采样时需要读取 **Pass 1** 的透射率图。

#### 4. Pass 3: 生成天空视图查找表 (Sky-View LUT)
*   **要做的事**：专为渲染“天空背景”准备的贴图。把当前相机所能看到的天空颜色预先画到一张全景图里。
*   **工程实现**：
    *   **目标**：一张分辨率适中的 2D 纹理（如 `200 x 100`）。
    *   **坐标映射**：关键工程细节——由于地平线处的颜色变化最剧烈，需要使用**非线性映射**（如二次曲线），把更多的纹理像素集中在地平线附近。
    *   **计算方式**：从当前相机位置出发向天空发射射线。沿射线进行步进（Ray-marching），在每个步进点读取 **Pass 1** 和 **Pass 2** 的纹理，计算阳光照射到这里并散射入相机的总能量。

#### 5. Pass 4: 生成空间透视体积图 (Aerial Perspective LUT)
*   **要做的事**：专为渲染场景中“地形、建筑、角色上的体积雾效”准备的体纹理。
*   **工程实现**：
    *   **目标**：一张贴合相机视锥体的 3D 纹理（Froxel，如 `32 x 32 x 32`）。RGB通道存散射光颜色（雾的颜色），Alpha通道存透射率（物体本身被遮蔽的程度）。
    *   **计算方式**：启动 3D Compute Shader，从相机出发，一层一层（Slice by Slice）向外计算。每一层都基于前一层的结果累加当前切片的散射和吸收。计算时同样需要采样 **Pass 1** 和 **Pass 2**。

#### 6. Pass 5: 最终场景合成 (Final Rendering / Application)
*   **要做的事**：在常规的前向渲染（Forward）或延迟渲染（Deferred）管线中，把前面算好的 LUT 应用到画面上。
*   **工程实现**：
    *   **画天空 (Skybox)**：在渲染管线的极早期或极晚期，画一个全屏四边形。根据屏幕像素对应的世界空间方向，去 **Pass 3** (Sky-View LUT) 中采样颜色直接输出。最后叠加上太阳圆盘（Sun Disk）。
    *   **画物体雾效 (Opaque/Transparent Fog)**：在物体的光照阶段结束后（比如Deferred管线的最后一步，或者半透明物体的材质中）。通过像素的深度值推算出它的世界坐标，然后用这个坐标去 **Pass 4** (3D体积纹理) 中进行三线性插值采样。
    *   **合成公式**：`最终像素颜色 = 物体原本颜色 * LUT.Alpha + LUT.RGB`。

### 工程落地的优势（为何采用此方案）：
1. **纯动态**：由于没有任何漫长的预计算，美术可以实时拖拽滑动条调整大气浓度、改变太阳角度，所见即所得。
2. **极高伸缩性**：在高端PC上可以调大LUT分辨率和Ray-marching步数，甚至引入阴影体积；在移动端（手机）上缩小LUT分辨率依然能得到非常平滑的结果。
3. **易于集成**：不管现有引擎是前向渲染还是延迟渲染，前置的4个 Compute Shader Pass 都能轻易地插入在主渲染流程之前。
